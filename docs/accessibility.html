Accessibility statement

This documentation is designed to conform to WCAG 2.1 Level AA.

Conformance target

Web Content Accessibility Guidelines (WCAG) 2.1, Level AA. The site is built as static HTML and remains fully readable and navigable with JavaScript, custom styling, or images disabled.

Measures taken

  • Semantic structure — landmark regions (header, nav, main, footer), a single h1 per page, and a logical heading order (WCAG 1.3.1, 2.4.6).
  • Skip link — a keyboard-reachable "skip to main content" link precedes the navigation (2.4.1).
  • Contrast — every text and background pair meets or exceeds the 4.5:1 ratio for normal text (3:1 for large text and UI components), verified in both light and dark themes (1.4.3, 1.4.11).
  • Visible focus — a high-contrast focus indicator is shown for every interactive element on keyboard focus (2.4.7).
  • Keyboard operable — all functionality, including the theme toggle, is reachable and operable by keyboard alone (2.1.1).
  • Responsive and reflowable — content reflows to a single column without horizontal scrolling down to 320 CSS pixels; wide code and tables scroll within their own container (1.4.10).
  • Respects user preferences — honors prefers-color-scheme and prefers-reduced-motion; the theme toggle's state is conveyed with aria-pressed (1.4.12, 2.3.3).
  • Language and labels — the page language is declared, tables carry captions and column headers, and navigation exposes the current page with aria-current.

Known limitations

Conformance is self-assessed and evaluated on an ongoing basis; it has not yet been audited by an independent third party. If you encounter a barrier, please open an issue on the project tracker so it can be addressed.

Feedback

Accessibility feedback is welcome and treated as a defect, not an enhancement. File it on the issue tracker linked above.
