PQC-Scout

An open post-quantum cryptography migration scanner: it inventories your cryptographic surfaces, scores which ones quantum computers break first, and hands you a standards-aligned migration roadmap.

The transition to post-quantum cryptography is now a regulatory obligation for hundreds of thousands of entities under NIS2, DORA, and the EU Cyber Resilience Act. Commercial assessment tools are vendor-locked and enterprise-priced. PQC-Scout is free, permissively licensed (Apache-2.0), and runs anywhere Python does.

What it does

  1. Enumerates cryptographic surfaces across ten infrastructure categories — TLS, code signing, storage encryption, key exchange, firmware integrity, API authentication, VPN tunnels, email signing, PKI hierarchy, and hardware keys.
  2. Scores each surface against a Q-Day horizon model aligned with NIST IR 8547 and Mosca's theorem, classifying exposure as CRITICAL, HIGH, MONITOR, or SAFE.
  3. Emits a phased migration roadmap citing NIST FIPS 203 (ML-KEM), 204 (ML-DSA), 205 (SLH-DSA), and CISA CNSA 2.0.
  4. Applies sector-aware retention priors — permanent data such as genomic, biometric, or identity credentials is modeled as an effectively unbounded confidentiality horizon.

Harvest-now, decrypt-later makes this a present-tense problem: traffic and identity data recorded today are decryptable the day a cryptographically relevant quantum computer arrives. A privacy guarantee with an unexamined quantum expiry date is a deadline nobody wrote down.

Try it in thirty seconds

pip install .
pqc-scout assess --target "ExampleCo" \
  --stack "nginx TLS 1.3, RSA-2048 code signing, AES-128 backups"

Project status

Milestone 1 (public source, standalone packaging, this documentation site) is complete. Subsequent milestones add deterministic config parsing, crypto-SBOM export (CycloneDX 1.6 + CSAF 2.0), supply-chain scanning, and CI integration. See the repository for the full roadmap.
